# 03 Planning System

Planning lives in:

- `GOAL.md`
- `docs/briefs/NNN-*.md`
- `docs/reviewer-messages/NNN-*.md`
- `docs/manager-log/NNN-*.md`

Each brief should define:

- objective;
- product value;
- acceptance criteria;
- expected files;
- validation commands;
- evidence to record;
- reachability or demo proof;
- out-of-scope work;
- stop conditions.

Keep briefs small. A brief should be finishable in one executor turn.
